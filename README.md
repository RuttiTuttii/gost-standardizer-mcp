# gost standardizer

локальный mcp-инструмент для проверки, стандартизации и навигации по gost-подобным требованиям в word-документах.

## что умеет

- анализировать `.docx`, `.docm` и `.doc`
- предлагать подходящий пресет оформления
- объяснять, почему был выбран именно этот пресет
- сравнивать документ с выбранным профилем и показывать структурированные отклонения
- собирать копию документа с единым шрифтом, полями, интервалами и абзацными отступами
- сохранять и загружать профили оформления в json
- искать актуальные документы и темы в локальном `meganorm`-каталоге и в живом источнике

## что внутри

- `plugins/gost-standardizer/.codex-plugin/plugin.json`
- `plugins/gost-standardizer/.mcp.json`
- `scripts/gost_standardizer.py`
- `scripts/mcp_server.py`
- `scripts/meganorm_catalog.py`

## базовая модель

в проекте есть 4 встроенных пресета:

- `report`
- `office`
- `technical`
- `legacy-college`

их можно использовать напрямую или как основу для собственных профилей в папке `profiles/`.

## новые tools

### `list_presets`

показывает встроенные пресеты и их назначение.

### `list_profiles`

показывает встроенные и сохранённые профили.

### `load_profile`

загружает профиль по имени или пути к json-файлу.

### `save_profile`

сохраняет профиль json-файлом на основе встроенного пресета.

### `inspect_document`

читает документ, угадывает подходящий пресет и возвращает:

- статистику
- список отклонений
- рекомендации
- образцы абзацев

### `explain_preset`

показывает, какие сигналы повлияли на выбор пресета:

- имя файла
- ключевые слова в тексте
- баллы по каждому пресету

### `validate_document`

проверяет документ относительно пресета или профиля и возвращает:

- severity
- rule id
- confidence
- auto-fixable
- evidence
- recommendation

### `compare_to_preset`

возвращает diff-подобное сравнение документа с выбранным профилем.

### `standardize_document`

создаёт новую стандартизированную копию с флагами:

- `fix_page_setup`
- `fix_styles`
- `fix_paragraphs`
- `fix_tables`
- `aggressive`

### `refresh_meganorm_cache`

обновляет локальный кэш `meganorm`.

### `search_meganorm_catalog`

ищет категории и документы по названию.

### `get_meganorm_topics`

возвращает текущие темы каталога или темы выбранной категории.

### `find_current_gost`

ищет только актуальные `ГОСТ` и `ГОСТ Р` записи с более точным матчингом по номеру.

## форматирование

по умолчанию проект использует:

- `Times New Roman`
- размер текста 14 pt
- межстрочный интервал 1.5
- левое поле 30 мм для отчётных документов
- правое поле 10 мм
- верхнее и нижнее поле 20 мм

## работа с `.doc`

если входной файл имеет расширение `.doc`, инструмент пытается конвертировать его в `.docx` через `soffice` / LibreOffice.

если конвертера нет, будет понятная ошибка с советом сначала пересохранить файл в `.docx`.

## ограничения

- строгую сертификацию соответствия ГОСТ проект не заменяет
- сложные колонтитулы, таблицы и секции всё равно лучше проверять визуально
- профили организации нужно подстраивать под реальные требования кафедры или компании

## требования

- python 3.12
- `python-docx`

установка зависимостей:

```bash
python -m pip install -r requirements.txt
```

## запуск mcp

сервер запускается через python-скрипт:

```bash
python scripts/mcp_server.py
```

`.mcp.json` уже указывает на этот вход.

## meganorm

источник берётся отсюда:

- [meganorm актуализированная база](https://meganorm.ru/mega_doc/norm/norm.html)

в ответах к записям всегда указывается происхождение:

- `origin=cache` значит запись взята из локального кэша
- `origin=source` значит запись получена живьём из нормативного html

для каталогов и документов также возвращается `updated_at`, чтобы было видно, когда кэш был актуализирован.
